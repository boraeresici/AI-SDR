# Security Policy

## Reporting a Vulnerability

Please do not disclose security issues publicly before triage.

Report with:

1. Affected component(s)
2. Reproduction steps
3. Impact assessment
4. Suggested mitigation (if available)

## Response Process

1. Acknowledge receipt.
2. Triage severity and scope.
3. Prepare fix and validation plan.
4. Coordinate disclosure after fix is available.

## Scope

Security reports can include:

- Skill prompt-injection risks
- Contract or auth misconfiguration risks
- Artifact leakage or sensitive data exposure
- Dependency and supply-chain risks in repo automation

## Disclosure Guidance

- Avoid posting exploit details in public issues.
- Share minimum required proof for reproduction.
- Wait for maintainer confirmation before broad disclosure.

## Support Window

Security fixes are prioritized for the latest `main` branch state.
